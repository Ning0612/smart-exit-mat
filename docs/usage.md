# 使用說明

## 開機流程

```
上電
  │
  ▼
載入 NVS 設定（ConfigManager::load）
  │
  ▼
初始化 LittleFS、秤（自動 tare）
  │
  ├─[WiFi SSID 為空]─→ 啟動 AP 模式（SmartExitMat-Setup，WPA2 密碼印於 Serial）
  │
  └─[有 SSID]─→ 嘗試連接 WiFi（最多 10 秒）
                   │
                   ├─[成功]─→ 啟動 Web Portal（STA 模式）
                   │          → NTP 校時
                   │          → 清理 6 個月前舊日誌
                   │          → 發送上線 LINE 通知
                   │          → 進入主迴圈
                   │
                   └─[失敗]─→ 啟動 AP 模式（SmartExitMat-Setup，WPA2 密碼印於 Serial）
```

---

## 主迴圈行為

主迴圈（`loop()`）每次迭代執行順序：

1. **處理 Web 請求**：`g_portal.handleClient()`（非阻塞）
2. **GPIO0 長按偵測**（若非 AP 模式）
3. **讀取 HX711 重量**：`g_scale.readWeightKg(3)`（3 次取樣，約 300 ms）
4. **更新踩踏狀態機**：`g_stepDetector.update()`
5. **處理踩踏事件**（若 eventFired）
6. **每日 NTP 同步**：`g_timeMgr.updateDailySync()`
7. **天氣更新**（僅在地墊閒置時）：`g_weather.updateIfNeeded()`

## 踩踏偵測狀態機

### 三態說明

```
              weight > stepOn
IDLE ─────────────────────────────→ MEASURING
 ▲                                      │
 │                                      │ 持續記錄最大重量
 │            weight < stepOff          │
 │  ┌─────────────────────────────── COOLDOWN ←─┤
 │  │  (cooldownMs 過後回 IDLE)                  │ 觸發事件：eventFired = true
 │  │                                             │ eventMaxWeightKg() 回傳最大值
 └──┘
```

### 各狀態說明

**IDLE（等待）**
- 地墊無人或讀值低於觸發門檻
- 可以執行 Web 請求、NTP 更新、天氣更新

**MEASURING（量測中）**
- 有人站在地墊上
- 持續記錄本次踩踏的最大重量值
- 離開時（重量低於 Step Off Threshold）觸發事件並進入 COOLDOWN

**COOLDOWN（冷卻）**
- 離開後的緩衝期，忽略所有重量變化
- 預設 1500 ms，可在設定頁調整
- 結束後回到 IDLE

### 為何記錄「最大重量」而非「離開時的重量」

踩踏過程中，體重讀值的時序大致如下：

```
kg
|        ████
|      ██████████
|    ████████████████
|  ██████████████████████
| ─────────────────────────→ 時間
  踩下    峰值    離開
```

離開時（進入 COOLDOWN 的瞬間）重量正在下降，此時讀值偏低，不能準確反映真實體重。最大值最接近靜止站立時的重量，與使用者登錄體重最吻合。

## 使用者識別邏輯

踩踏事件觸發後：

1. 取得本次踩踏最大重量 `eventWeight`
2. 與所有登錄使用者的體重做差值比較
3. 找出最小差值 `bestDiff`（最佳配對）與第二小差值 `secondBestDiff`
4. 若 `bestDiff > matchToleranceKg`：判定為**未知使用者**
5. 否則：識別為最佳配對使用者

**識別不明確保護**：若 `secondBestDiff - bestDiff < 1.5 kg`，識別結果不確定（體重過於接近），體重自學習機制會跳過本次更新，但出門/回家狀態仍正常切換。

---

## 出門／回家狀態切換

每次成功識別後，`StateManager::toggle()` 交替切換該使用者的 `atHome` 狀態：

```
atHome=false → 踩踏 → atHome=true  → LINE 通知「回家了」
atHome=true  → 踩踏 → atHome=false → LINE 通知「出門了」
```

狀態儲存於 NVS `state` 命名空間，斷電後保留。

**初始狀態**：新登錄使用者預設 `atHome=false`（出門中）。

---

## 體重自適應學習

系統在每次已知使用者踩踏後，自動判斷是否需要更新登錄體重。

### 觸發條件

以下**所有條件**同時滿足才會更新：

1. 識別成功（非未知使用者）
2. 識別結果明確（與第二候選差距 ≥ 1.5 kg）
3. 偵測重量與登錄重量差距超過 `matchToleranceKg × 0.5`
4. 連續 3 次偵測結果彼此接近（一致性驗證）
5. 新體重在 20.0–200.0 kg 的合理範圍內

### 更新機制

```
第 1 次偏差（> tolerance×0.5）→ 記錄累積
第 2 次偏差（與前次接近）    → 繼續累積
第 3 次偏差（與前次接近）    → 計算 3 次平均值 → 更新體重 → 寫入 NVS
```

若任何一次偏差方向改變或讀值不一致（差距 > toleranceKg），重置累積器。

### 效果

使用者體重因季節、飲食等因素自然變化時，系統可自動跟隨更新，維持識別準確度。

## 未知使用者處理

重量超過 20 kg 但無法配對任何登錄使用者時：

1. **記錄事件日誌**（`uid="0"`, `nm="Unknown"`, `ev="unknown"`）—不依賴 LINE 冷卻
2. **LINE 警示通知**：每 3 分鐘最多一次（避免短時間內多次踩踏淹沒通知）

20 kg 門檻的作用：過濾地墊上的輕微振動或小型物品，只有確定是人踩上才發送警示。

## 硬體重置（清空 WiFi）

當 WiFi 設定錯誤、忘記 IP 或網路環境改變時，可透過硬體方式強制進入 AP 設定模式：

### 操作步驟

1. 長按 **BOOT 鍵（GPIO0）**，持續超過 3 秒
2. Serial 顯示提示：`[Runtime] GPIO0 long press — release button to clear WiFi and enter AP mode`
3. **放開按鍵**（重要：必須放開後才重啟，以確保 GPIO0 回到 HIGH，避免進入 bootloader download 模式）
4. 裝置重啟，清空 WiFi SSID 和密碼，進入 AP 熱點模式

> 此操作**只清空 WiFi 憑證**，其他設定（LINE Token、使用者、狀態、事件日誌）均保留不受影響。

### 在 WiFi 連線等待期間也可觸發

若開機時 WiFi 連接中（最多等 10 秒），在這段等待期間同樣可長按 BOOT 鍵觸發重置。

## 存取 Web 介面

### 取得裝置 IP

方法一（推薦）：開機成功後，LINE 通知訊息中包含：
```
智慧地墊已上線
區網設定頁：http://192.168.x.x
（需與地墊在同一 WiFi 下）
```

方法二：登入路由器後台，在「已連線裝置」或「DHCP 用戶端列表」中尋找名稱為 `ESP32` 的裝置。

方法三：Serial 監控中會顯示：
```
[WiFi] Connected, IP: 192.168.x.x
```

### 頁面說明

| URL | 內容 |
|-----|------|
| `http://<IP>/` | 設定頁：修改所有設定 |
| `http://<IP>/dashboard` | 儀表板：即時天氣、在家狀態、事件報表 |

儀表板功能：
- **即時天氣**：溫度、體感溫度、天氣建議
- **使用者狀態卡片**：每位使用者目前在家/出門
- **日報表**：指定日期的出門/回家事件時序
- **週報表**：本週（或指定週）每天的出入記錄
- **月報表**：指定月份的統計
- **12 小時時間軸**：過去 12 小時各使用者狀態變化

## 天氣建議邏輯

出門事件發生時，`WeatherManager::advisory()` 提供以下建議：

### 當前天氣（即時）

| 天氣代碼 | 描述 | 建議文字 |
|---------|------|---------|
| 200–299 | 雷電 | 正在打雷閃電，請注意安全 |
| 300–399 | 毛毛雨 | 正在飄毛毛雨，記得帶傘 |
| 500–599 | 雨 | 正在下雨，記得帶傘 |
| 600–699 | 雪 | 正在下雪，注意保暖 |
| 氣溫 < 12°C | — | 天氣寒冷，記得穿外套保暖 |
| 氣溫 12–16°C | — | 天氣涼，建議多穿一件 |
| 體感 > 35°C | — | 天氣炎熱，注意防曬補水 |

### 12 小時預報（若當前無雨）

| 條件 | 建議文字 |
|------|---------|
| 未來有雷雨（pop > 30%） | 12 小時內可能有雷雨，謹慎出行 |
| 未來有雨（pop > 40%） | 待會可能下雨（機率 XX%），建議帶傘 |

多個建議時，以換行符（`\n`）串接，LINE 訊息中顯示為多行。
